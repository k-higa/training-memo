# 筋トレ管理Webサービス 要件定義書

## 1. プロジェクト概要

### 1.1 サービス名
**Training Memo** - 筋トレ記録・管理アプリケーション

### 1.2 目的
日々の筋力トレーニングを記録・管理し、進捗を可視化することで、継続的なトレーニングをサポートする。

### 1.3 ターゲットユーザー
- 筋トレを習慣化したい初心者
- トレーニング記録を効率的に管理したい中級者〜上級者
- パーソナルトレーナーとクライアント

---

## 2. 機能要件

### 2.1 ユーザー管理機能

| 機能 | 説明 | 優先度 |
|------|------|--------|
| ユーザー登録 | メールアドレス・パスワードでの新規登録 | 高 |
| ログイン/ログアウト | セッション管理 | 高 |
| プロフィール設定 | 身長・体重・目標などの設定 | 中 |
| パスワードリセット | メールによるパスワード再設定 | 中 |
| ソーシャルログイン | Google/Apple IDでのログイン | 低 |

### 2.2 トレーニング記録機能

| 機能 | 説明 | 優先度 |
|------|------|--------|
| トレーニング登録 | 日付・種目・重量・回数・セット数を記録 | 高 |
| トレーニング編集 | 登録済みの記録を編集 | 高 |
| トレーニング削除 | 登録済みの記録を削除 | 高 |
| クイック入力 | 前回の記録をコピーして入力 | 中 |
| メモ機能 | トレーニングにメモを追加 | 中 |
| 写真添付 | フォームチェック用の写真を添付 | 低 |

### 2.3 種目マスタ管理

| 機能 | 説明 | 優先度 |
|------|------|--------|
| 種目一覧表示 | プリセット種目の一覧表示 | 高 |
| カスタム種目登録 | ユーザー独自の種目を追加 | 中 |
| 部位別フィルタ | 胸・背中・脚などで絞り込み | 中 |
| お気に入り種目 | よく使う種目をお気に入り登録 | 低 |

### 2.4 メニュー管理機能

| 機能 | 説明 | 優先度 |
|------|------|--------|
| メニュー作成 | トレーニングメニューを作成・保存 | 中 |
| メニューテンプレート | 曜日別のルーティンを設定 | 中 |
| メニュー共有 | 他ユーザーとメニューを共有 | 低 |

### 2.5 進捗管理・分析機能

| 機能 | 説明 | 優先度 |
|------|------|--------|
| カレンダー表示 | トレーニング実施日をカレンダーで確認 | 高 |
| 履歴一覧 | 過去のトレーニング記録を一覧表示 | 高 |
| グラフ表示 | 重量・回数の推移をグラフ化 | 高 |
| 部位別集計 | 部位ごとのトレーニング頻度を表示 | 中 |
| 自己ベスト記録 | 種目ごとの最高記録を表示 | 中 |
| 週間/月間レポート | 期間ごとのサマリーレポート | 低 |

### 2.6 体重・体組成管理

| 機能 | 説明 | 優先度 |
|------|------|--------|
| 体重記録 | 日々の体重を記録 | 中 |
| 体脂肪率記録 | 体脂肪率を記録 | 低 |
| 体重推移グラフ | 体重の変化をグラフ表示 | 中 |

## 3. 非機能要件

### 3.1 パフォーマンス
- ページ読み込み時間: 3秒以内
- API応答時間: 500ms以内
- 同時接続ユーザー数: 1,000人以上

### 3.2 セキュリティ
- HTTPS通信の必須化
- パスワードのハッシュ化保存（bcrypt）
- JWT認証によるセッション管理
- SQLインジェクション・XSS対策
- CORS設定

### 3.3 可用性
- 稼働率: 99.5%以上
- 定期バックアップ（日次）
- 障害時の復旧時間: 4時間以内

### 3.4 対応デバイス
- レスポンシブデザイン（モバイルファースト）
- 対応ブラウザ: Chrome, Safari, Firefox, Edge（最新2バージョン）
- PWA対応（オフライン機能）

## 4. データモデル（概要）

### 4.1 主要エンティティ

```
User (ユーザー)
├── id
├── email
├── password_hash
├── name
├── height
├── created_at
└── updated_at

Exercise (種目マスタ)
├── id
├── name
├── muscle_group (部位)
├── is_custom
├── user_id (カスタム種目の場合)
└── created_at

Workout (トレーニング記録)
├── id
├── user_id
├── date
├── memo
└── created_at

WorkoutSet (セット記録)
├── id
├── workout_id
├── exercise_id
├── set_number
├── weight
├── reps
└── created_at

BodyWeight (体重記録)
├── id
├── user_id
├── date
├── weight
├── body_fat_percentage
└── created_at
```
